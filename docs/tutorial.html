装好之后，先做这几件事

先体验一次真正有结果的使用，再决定要不要继续深入技能、渠道和自动化。

📖 建议顺序

  1. 先完成第一次对话
  2. 再装一个对你有用的技能
  3. 需要时再接消息渠道
  4. 记忆系统
  5. 定时任务
  6. 配置管理

1. 先完成第一次对话

你现在最需要的是成就感，不是把所有功能都学完

ClawStart 跑通后，底层的 OpenClaw 环境就已经可用了。你现在可以直接在对话框输入任何问题或指令，让它开始真正帮你做事。

# 问问题 帮我解释一下 Docker 和虚拟机的区别 # 执行任务 帮我把 ~/Downloads 里超过 30 天的文件整理到 ~/Archive # 写代码 用 Python 写一个批量重命名文件的脚本
💡 第一轮目标 先确认它能稳定响应、能理解你的指令、能给出你满意的第一条结果。这样你就已经跑通了最关键的一步。

2. 再装一个对你有用的技能

不要一上来装十几个，先选一个你马上就会用到的

# 搜索可用技能 openclaw skills search weather # 安装技能 openclaw skills install weather # 查看已安装技能 openclaw skills list

安装后直接在对话中使用，AI 会自动调用：

# 安装了 weather 技能后 北京今天天气怎么样？ # 安装了 git-essentials 技能后 帮我看看这个仓库最近的提交记录
💡 技能选择建议 第一次优先选搜索、文件整理、Git 或写作类技能。装一个、用一次，比收藏二十个更有效。

3. 需要时再接消息渠道

这一步不是首装必做，等你确认它值得继续用，再接到聊天工具里

💬 Telegram

通过 BotFather 创建机器人，填入 Token 即可连接。支持私聊和群组。

🎮 Discord

创建 Discord Bot，配置 Token 和频道权限。支持多服务器。

📱 微信（实验性）

通过扫码登录网页版微信，实现消息收发。稳定性取决于微信接口。

🐧 QQ

通过 QQ 官方机器人 API 接入，支持群聊和私聊。

4. 记忆系统

AI 助手会记住你说过的话，越用越懂你

底层的 OpenClaw 有分层记忆系统。短期记忆在对话上下文中，长期记忆写入本地文件。数据始终在你自己的设备上。

# 让 AI 记住某件事 记住：我的服务器 IP 是 192.168.1.100，SSH 端口是 2222 # 之后直接引用 帮我 SSH 到我的服务器看看磁盘使用情况
💡 记忆文件 长期记忆保存在 ~/.openclaw/workspace/MEMORY.md，你可以随时查看和编辑。

5. 定时任务

让 AI 在指定时间自动执行任务

# 创建定时任务 openclaw cron add --schedule "0 9 * * *" --task "检查今天的日历安排并推送到 Telegram" # 查看所有定时任务 openclaw cron list # 删除定时任务 openclaw cron remove <task-id>

6. 配置管理

调整 AI 的行为、模型和系统设置

# 查看当前配置 openclaw config list # 修改默认模型 openclaw config set defaultModel "anthropic/claude-sonnet" # 查看运行状态 openclaw status
💡 个性化 编辑 ~/.openclaw/workspace/SOUL.md 可以定义 AI 的性格和行为风格。

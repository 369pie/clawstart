按人群选 Skills

如果你不想先研究一堆分类，就按你自己的工作类型来选。第一次先装最像你日常任务的那一类。

先记一个原则 第一次不是挑“最强”的 Skill，而是挑“最像你每天都在做的事”的 Skill。

内容用户

适合创作者、运营、学生

第一优先：写作 / 内容处理类

适合润色文案、生成提纲、改写不同平台版本，最容易直接看到产出。

最适合

第二优先：搜索 / 查询类

适合查资料、补背景、快速验证信息，能明显提升写作和选题效率。

常用

办公用户

适合上班族、项目协作、日常整理

第一优先：文件 / 整理类

适合会议纪要、待办提取、资料梳理，最容易直接省时间。

最稳

第二优先：搜索 / 查询类

适合做背景资料补充、外部信息确认和快速查证。

高频

开发用户

适合开发者和半技术用户

第一优先：Git / 仓库类

适合看最近提交、理解仓库结构、快速做代码复盘，装前装后差异很明显。

最直观

第二优先：搜索 / 查询类

适合补文档、查资料、对比方案，让技术研究和排查更顺手。

辅助

怎么做决定

如果你横跨多个角色，就按最近一周最常做的事来选

  • 最近最常写东西：先装写作 / 内容处理类
  • 最近最常开会、整理资料：先装文件 / 整理类
  • 最近最常看代码、看仓库：先装 Git / 仓库类
  • 如果你三类都做，那就先装搜索 / 查询类作为通用起点

按人群选，是为了少走弯路

你不需要先学完整 Skill 生态。只要先找到最像你日常任务的那一类，就已经比盲装一堆更有效。

ClawStart Skills 指南

Skills 指南

第一次只装一个最有用的 Skill。不要一开始收藏一堆，先装上、先用起来、先确认它真有价值。

这页适合谁 适合已经完成首启、能够正常对话，但想让 ClawStart 具备更明确能力的用户。

先理解一个概念

什么是 Skill

你可以把 Skill 理解成“给 AI 助手额外安装的能力包”。装了 Skill 之后，它不只是会聊天，还能更稳定地完成某类动作，比如天气查询、Git 操作、文件整理、内容处理。

搜索类 Skill

适合想让它查天气、查资料、查某类外部信息的用户。

工作类 Skill

适合写作、内容整理、办公总结、团队协作这些高频任务。

开发类 Skill

适合看 Git、跑检查、分析代码和整理技术资料的用户。

第一步

先选一个马上就会用到的 Skill

第一次不要装十个，先装一个。推荐优先顺序：搜索类、文件类、Git 类、写作类。

🌦️

搜索 / 查询类

适合第一次感受“装了 Skill 以后有什么变化”的用户，反馈最直观。

🗂️

文件 / 整理类

适合办公和内容用户，最容易形成“以后我会继续用”的复用感。

第二步

用最短路径安装第一个 Skill

# 推荐先在 Skills Hub 里确认 slug npx clawhub@latest install weather # 也可以先装百度搜索类 Skill npx clawhub@latest install baidu-search # 想装仓库类 Skill 时 npx clawhub@latest install github
成功标准：安装命令执行完成后，你能立刻在对话里用一句自然语言验证它。

第一次不建议盲搜。先去 Skills Hub 看中文分类和真实 Skill 页面，再回来执行安装命令会更稳。

先去 Skills Hub 看小白起步项 → 安装第一个 Skill →

第三步

装好以后立刻用一次

安装成功后不要停在“我已经装上了”，直接用一次，才能判断这个 Skill 是不是值得保留。

# 安装了 weather Skill 以后 北京今天天气怎么样？ # 安装了 git 相关 Skill 以后 帮我看看这个仓库最近 5 次提交都做了什么

常见问题

第一次装 Skill 最容易卡在这几类地方

搜得到，但装不上

通常是网络、权限、依赖或 registry 问题，优先去排障页看 Skill 安装失败相关部分。

去排障 →

装上了，但不会用

先从最短问题模板开始，不要一上来写很长的复杂任务。

看新手页 →

装太多，不知道保留哪个

优先保留你一周内会反复用到的，其他先删掉或暂时不用。

看资源 →

新手推荐

第一次建议优先装这 4 类

  • 搜索 / 查询类：最容易立刻验证效果
  • 文件整理类：适合日常办公和内容用户
  • Git / 仓库类：适合开发者和半技术用户
  • 写作 / 内容处理类：适合创作者和运营用户
看新手推荐 Skills → 按内容 / 办公 / 开发选 → 进入 Skills Hub →
真实目录 中文分类

如果你不想靠关键词硬猜，直接进 Skills Hub

我们已经把 ClawHub 上的公开 Skill 按中文场景重新分了类，并保留真实安装命令和原始 Skill 页面链接。第一次选 Skill，先从这里看最省时间。

打开新手预筛选 → 打开完整 Skills Hub → 查看原始 ClawHub →

装 Skill 的目标不是“变多”，而是“变好用”

第一次只要装一个真正帮你省时间的 Skill，就已经比“收藏很多但不用”更有价值。

安装第一个 Skill → 看推荐清单 → 按人群选 Skills → 继续看进阶使用 → 安装失败？去排障 → 了解 LolaClaw →

继续阅读

从“知道 Skill 是什么”走到“真的装上第一个”

这页解决的是认知问题。接下来最值钱的，不是继续看概念，而是按推荐顺序选一个最适合你的 Skill 并把它装上。

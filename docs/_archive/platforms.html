按平台选择

先按你的系统和目标分流，这比盲点下载有用多了。

Windows、macOS、Linux 面对的坑位不同，用户目标也不同。有人只想今天先跑通一次，有人需要更稳的首装路径，也有人已经在考虑多机迁移和更省事的长期体验。这个页面的任务，就是把你送到对的下一步。

一句话建议：先按平台选，再按目标选，不要把所有人都塞进同一条入口里。

先看你的系统，别先看别人的报错

平台差异不是配角。很多翻车并不神秘，只是你拿错了入口。

Windows

最常见，也最容易被权限、路径和环境变量绊一下

如果你用的是 Windows，最值钱的不是猛冲安装，而是先把准备动作做干净。这个平台最容易出现的，不是神秘 bug，而是路径、权限、终端和环境没对齐。

  • 第一次安装前，优先看安装前准备和检查清单
  • 如果你今天就要装起来，还是先走官方路径
  • 一旦出错，别同时换目录、权限和安装方法
进入 Windows 专属入口 先做安装前准备 再过安装检查清单
macOS

整体更顺一点，但别因为看起来体面就放松警惕

macOS 往往首装体验会更平滑一点，但这不等于可以跳过顺序。权限提示、系统限制、目录选择这些事，照样能把第一次上手变成一次很有艺术感的翻车。

  • 大多数用户可直接按快速开始进入官方路径
  • 如果你怕自己漏步骤，先过检查清单更稳
  • 已经能跑通一次的人，再考虑迁移和便携
进入 macOS 专属入口 先核对安装清单 再去快速开始
Linux

弹性最大，也最容易对自己的系统做出一些富有创造力的伤害

Linux 用户通常更能接受命令行，但问题在于，你也更容易觉得“这个我自己改一下就行”，然后把简单问题改成悬疑片。第一轮仍然建议先走最短路径，先确认能跑，再谈自定义。

  • 适合直接看下载与安装和官方文档
  • 出错时优先保留原始错误，不要急着自创修复流派
  • 如果要长期多机使用，再去看便携和进阶方案
进入 Linux 专属入口 去下载与安装 先看排障思路

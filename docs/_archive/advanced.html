⚡ 高级能力

多模型切换与路由

根据任务类型自动选择最合适的模型。代码任务用编程模型，日常对话用通用模型，翻译用专业模型。

  • 配置多个模型供应商，按场景自动路由
  • 设置 fallback 机制，主模型不可用时自动切换
  • 按成本和速度优化模型选择策略
# 在 openclaw.json 中配置模型路由 { "models": { "default": "anthropic/claude-sonnet", "coding": "volcengine/ark-code-latest", "translation": "deepseek/deepseek-chat" } }
🤝 团队协作

多 Agent 协作

让多个 AI Agent 分工合作，像一个团队一样完成复杂项目。一个负责写代码，一个负责测试，一个负责文档。

  • 定义 Agent 角色和职责
  • Agent 之间可以互相通信和协调
  • 任务自动分配和进度追踪
💡 实际案例 ClawForge Studio 就是一个多 Agent 开发团队的实践：产品经理 Agent 拆解需求，开发 Agent 写代码，测试 Agent 验证质量，全程自动推进。
🛠️ 开发者

自定义技能开发

为 OpenClaw 开发专属技能，扩展它的能力边界。技能就是一个 SKILL.md 文件加上可选的脚本。

  • 用 Markdown 定义技能的行为规则
  • 可选：附带 Python / Node.js / Shell 脚本
  • 发布到 ClawHub 与社区共享
# 创建一个新技能 mkdir ~/.openclaw/workspace/skills/my-skill cat > ~/.openclaw/workspace/skills/my-skill/SKILL.md << 'EOF' --- name: my-skill description: 我的自定义技能 --- # 技能说明 当用户要求 XXX 时，执行以下步骤... EOF
⚡ 高级能力

自动化工作流

用定时任务和事件触发器构建自动化工作流。让 AI 在后台持续工作，不需要你手动触发。

  • 定时检查邮件、日历、社交媒体
  • 监控服务器状态，异常时自动告警
  • 定期整理文件、备份数据、生成报告
# 每天早上 9 点推送日程摘要 openclaw cron add \ --schedule "0 9 * * *" \ --task "检查今天的日历，总结待办事项，推送到 Telegram" # 每小时检查服务器状态 openclaw cron add \ --schedule "0 * * * *" \ --task "SSH 到服务器检查 CPU 和内存，超过 90% 就告警"
🏢 企业级

企业部署方案

在团队或企业环境中部署 OpenClaw，支持多用户、权限管理、审计日志。

  • 多用户独立工作区，数据隔离
  • 统一的模型配置和成本管控
  • 操作审计日志，满足合规要求
  • 私有化部署，数据不出内网
💡 需要帮助？ 企业部署涉及较多定制化配置。欢迎通过 GitHub Issues 提交需求，或加入 Discord 与社区交流。

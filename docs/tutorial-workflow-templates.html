工作流模板

当你已经会基础操作，下一步最该做的不是再学一堆新概念，而是把高频动作固定成重复可用的模板。

适合谁 适合已经完成第一次任务、已经知道 ClawStart 有价值，但希望它更稳定帮你省时间的用户。

先记一个原则

工作流不是“更复杂”，而是“更少重来”

如果一件事你一周内会重复两次以上，它就值得做成模板。这样你不需要每次重新想提问方式，也更容易看出效果有没有在变好。

办公 / 内容类

最容易先做成模板的 3 件事

模板 1：会议纪要 -> 待办清单

把会议记录整理成“结论、待办、负责人、截止时间”四段式输出。

最稳

模板 2：长文 -> 三段摘要

任何长内容都统一拆成“核心观点、风险点、下一步建议”。

高频

模板 3：一稿多用

把同一份内容改写成公众号、小红书、B 站简介等不同版本。

内容复用

开发 / 研究类

适合半技术用户和开发者的 3 件事

模板 4：仓库变更速览

固定让它总结最近 5 次提交，输出“做了什么、可能影响什么、我接下来该看哪”。

Git

模板 5：报错 -> 排查顺序

每次都按“现象、可能原因、先查哪三步”的格式解释报错，避免越修越乱。

排查

模板 6：资料阅读 -> 决策摘要

把长文档统一压成“结论、限制、是否值得继续”的决策卡片。

研究

落地方式

把工作流固定下来，只需要这 3 步

1

找出你最近一周重复出现的任务

不要从想象开始，而是从你已经在做的事情开始。

2

固定输出格式

把每次都要的格式写死，比如“结论 / 风险 / 建议”或“任务 / 负责人 / 截止时间”。

3

每周重复两三次，再决定要不要自动化

先跑顺手动版本，再决定值不值得交给自动化或更省心的 LolaClaw。

你不需要一开始就拥有完整系统

只要先把一个高频任务固定下来，ClawStart 就已经开始从“偶尔能用”变成“真的会帮你省时间”。

新手推荐 Skills

第一次装 Skill 不看数量，看你能不能真的用起来。优先从最直观、最常用、最容易验证效果的类型开始。

先记一个原则 第一次只装 1 到 2 个。能反复用，比装很多更重要。

推荐顺序

新手优先装这 4 类

🌦️

搜索 / 查询类

优先级最高。装完后最容易立刻问一句话就看到效果。

🗂️

文件 / 整理类

适合办公和内容用户，最容易在日常任务里重复使用。

🧰

Git / 仓库类

适合开发者或半技术用户，能很快看出“装前和装后”的区别。

✍️

写作 / 内容处理类

适合创作者和运营用户，最容易直接转化成实际产出。

怎么选

按你最近一周最常做的事来选

最近经常查信息

优先选搜索 / 查询类，最容易用一句自然语言就验证效果。

最推荐

最近经常整理文件或文稿

优先选文件 / 内容整理类，最容易感受到“它真的帮我省时间”。

高频

最近经常看仓库、改代码

优先选 Git / 代码相关类，能快速放大开发场景下的价值。

开发

不建议一开始就装的

这些先别急

  • 名字很抽象、看不出到底做什么的 Skill
  • 需要复杂前置依赖或很多额外配置的 Skill
  • 你最近根本不会用到的 Skill
  • 同类型装很多个，只会让你更难判断哪个值得保留

推荐清单的价值，不是帮你装更多

它的目标是帮你少走弯路，第一次就装一个最容易看到价值的 Skill。

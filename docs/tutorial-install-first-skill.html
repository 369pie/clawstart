安装第一个 Skill

第一次不要追求装很多。先选一个马上能用到的 Skill，完整走通搜索、安装、验证这条链路。

这页适合谁 适合已经能正常对话、想第一次安装 Skill，但不知道从哪开始的用户。

先做一个选择

选一个你一周内会真正用到的 Skill

  • 想快速看到结果：优先选搜索 / 查询类
  • 想提高办公效率：优先选文件 / 内容整理类
  • 想辅助开发：优先选 Git / 代码相关类

步骤 1

先搜索，再判断

# 搜索某类 Skill openclaw skills search weather # 或按关键词搜索 openclaw skills search git # 搜索文件整理类 openclaw skills search file
判断原则 第一次优先选“名字直白、用途明确、你今天就能试一次”的 Skill，不要先装很复杂的组合型 Skill。

步骤 2

只装一个

# 示例：安装 weather openclaw skills install weather # 安装后查看列表 openclaw skills list
成功标准：`skills list` 能看到它，说明安装阶段已经完成。

步骤 3

立刻验证一次

装完不要停在“我已经装上了”。马上提一个最短问题，验证这个 Skill 是不是真的能帮到你。

# 安装了 weather 以后 上海明天会下雨吗？ # 安装了 git 以后 帮我看一下这个仓库最近 3 次提交的重点

步骤 4

如果失败，优先看这 3 类问题

搜索结果有，但安装失败

通常是网络、权限、registry 或依赖问题，优先去排障页查 Skill 安装失败类问题。

去排障 →

安装成功，但不会调用

先把提问缩短、缩具体，优先问一个最容易判断成功与否的问题。

回新手入门 →

装完发现不适合自己

不用纠结，第一次试错很正常。回去看推荐清单，换一个更直接的 Skill 即可。

看推荐清单 →

第一次装 Skill 的目标不是“装成专家”

你只需要完成一次搜索、一次安装、一次验证，这就已经比“知道有 Skill 但从没动手”更进一步了。

OpenClaw API 配置指南

第一次先跑通连接，再谈最优配置

如果你搜的是“OpenClaw API Key 怎么配”，先不要陷进模型大战。第一次优先选最容易稳定连接的路径。

推荐顺序 中国大陆普通用户第一次优先走“国内在线模型 → 已有海外 API 的模型 → 本地模型”。目的是先稳定跑通，而不是一开始追求最强。

第一步

先判断你现在最适合哪条模型路径

🇨🇳

国内在线模型

适合大多数中国大陆用户。优先用于第一次配置和连通性验证，通常更省心。

🌍

海外在线模型

适合已经有稳定账号和余额的用户。重点先确认当前网络和账单状态正常。

🏠

本地模型

适合第二阶段再尝试。第一次主路径不推荐从这里开始，否则很容易在环境问题里打转。

第二步

API Key 配置时先检查这 4 件事

  • 复制时有没有带上前后空格
  • 当前 Key 是否有效、是否还有余额或额度
  • 你选择的模型服务商和填入的 Key 是否对应
  • 当前网络是否能访问对应接口

第三步

第一次成功标准，不是“模型最强”，而是“连接稳定”

1

测试连接能稳定通过

2

能发出一个问题并得到正常回复

3

你知道当前这条模型路径后面还能继续用

什么时候该直接去排障 如果你已经明确服务商和 Key 没问题，但还是测试失败、超时、报网关错误，就直接去 排障页，不要无穷切换模型服务商。

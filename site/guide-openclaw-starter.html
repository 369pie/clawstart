OpenClaw 中文入门手册

第一次从下载到会用，先把这 5 步走完

这是一份网页版中文入门手册。你可以把它当成未来 PDF 的母版，先按这一页走完“安装、配置、第一次任务、第一批 Skill”四条主路径。

这份手册适合谁 适合第一次接触 OpenClaw、刚装好还不确定下一步做什么、或想把零散教程压成一条完整学习链路的人。

第 1 步

先判断你的安装路径

普通用户优先走 Windows 或 macOS 一键包，不要第一步就陷进旧版命令行教程。Linux 当前主推荐路径是 Beta 安装脚本。

看安装指南 → 直接去下载页 →

第 2 步

把模型和 API 先配稳定

第一次目标不是“模型最强”，而是“连接稳定”。中国大陆用户优先走国内在线模型，再考虑已有海外 API 或本地模型。

看 API 配置指南 → 看完整模型教程 →

第 3 步

完成第一次真实任务

不要停在“已经装好了”。你需要立刻完成一次写作、办公或开发小任务，这样才算真正开始用起来。

照抄任务模板 → 看新手入门 →

第 4 步

只装第一个最像你日常任务的 Skill

第一次不要装很多。优先从“适合小白”“无需 API Key”或者最像你日常场景的 Skill 开始。

看中文 Skills 指南 → 先看新手 Skill 入口 →

第 5 步

按你的身份扩展，而不是盲学全部功能

内容用户、办公用户、开发者和研究学习用户，最适合的下一步并不一样。先按自己的身份走对应路径，比“把所有教程看完”更有效。

✍️

内容创作者

写作、改稿、资料总结和内容沉淀，优先走更偏创作产出的路径。

进入创作者路径 →
💼

办公用户

会议纪要、待办清单、长文档总结和协作资料，优先走办公提效路径。

进入办公路径 →
💻

开发者

报错解释、仓库阅读、Issue / PR 整理和 GitHub Skill，优先走开发路径。

进入开发者路径 →
🔬

研究学习用户

长文资料总结、决策摘要和知识整理，优先走研究学习路径。

进入研究路径 →

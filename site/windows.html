Windows 一键包步骤

  1. 下载安装包

    打开最新版本页面，选择 Windows 对应的压缩包下载。第一次安装优先用一键包，最简单省心。

  2. 解压到新目录后再运行启动器

    先把压缩包完整解压到一个新的空目录，再双击启动程序（launch.bat）。如果 SmartScreen（Windows 自带的安全拦截）弹窗，点击「更多信息」→「仍要运行」即可。请先完整解压再运行，也不要覆盖旧目录。

  3. 完成首次引导并拿到第一次正常回复

    按引导选择模型服务商、填入 API Key（服务商给你的专属密码）、测试连通性。浏览器打开后，先确认你已经拿到一条正常回复，这才算真的跑通。

高级：命令行安装步骤

💡 适合谁 以下步骤面向熟悉 Node.js 编程环境的开发者。如果你是普通用户，请直接使用上面的一键包。

  1. 安装 Node.js

    OpenClaw 需要 Node.js 22 或更高版本。去 nodejs.org 下载 LTS 版本安装。

    或者用包管理器安装：

    # Winget (推荐) winget install OpenJS.NodeJS.LTS # Scoop scoop install nodejs-lts # Chocolatey choco install nodejs-lts

  2. 全局安装 OpenClaw

    npm install -g openclaw@latest

    也可以用 pnpm：pnpm add -g openclaw@latest

  3. 运行引导向导

    openclaw onboard --install-daemon

    向导会引导你完成 Gateway、Workspace、频道和 Skill 的配置。

推荐环境与已知问题

✅ 推荐环境

  • 系统版本：Windows 10 64 位 或 Windows 11（推荐最新正式版）
  • 内存：8GB 以上（推荐 16GB）
  • 磁盘空间：至少预留 2GB 可用空间
  • 网络：需要能访问 GitHub 和所选模型服务商 API
  • 浏览器：Chrome / Edge / Firefox 最新版

⏱️ 首次启动耗时预期

  • 解压一键包：约 10-30 秒（取决于硬盘速度）
  • 首次启动：约 30-60 秒（首次需要下载必要组件）
  • 浏览器打开：启动器会自动打开配置界面。如果没有自动打开，手动在浏览器地址栏输入 http://127.0.0.1:18789

⚠️ 已知问题

「系统无法接受输入的时间」错误

部分中文 Windows 环境下，启动器内部调用 timeout 命令时可能触发此错误。我们已在最新版本中修复为 PowerShell Start-Sleep，如仍遇到此问题：

  1. 确认已下载最新 Release 版本
  2. 尝试手动运行 diagnose.bat 查看详细日志
  3. 前往 排障页 搜索此错误码
logs 目录下没有日志文件

如果启动失败但 logs 目录为空，可能是网关未成功启动。请运行 diagnose.bat 进行诊断，日志会输出到控制台。

浏览器打开但显示无法连接

确认网关已完全启动（看到「OK OpenClaw 已启动」提示），并访问正确端口 18789 而非 3000。

常见问题

Windows SmartScreen 阻止安装？

弹出"Windows 已保护你的电脑"时，点击「更多信息」→「仍要运行」即可。OpenClaw 是开源软件，源码可在 GitHub 查看。

安装后运行报错 "找不到 openclaw 命令"？

关闭并重新打开命令提示符/ PowerShell，让 PATH 环境变量生效。如果仍然找不到：

# 检查 npm 全局路径 npm config get prefix # 将该路径添加到 PATH 环境变量 setx PATH "%PATH%;%APPDATA%\npm"
杀毒软件误报病毒？

部分杀毒软件可能会误报。你可以：

  • 将当前启动器文件夹添加到信任列表/白名单
  • 或者使用命令行版本（npm 安装）
  • 从 GitHub 下载源码自行编译

装好以后别停在“能打开了”

下一步不是继续研究安装，而是按快速开始完成模型测试和第一次正常输出。能拿到第一条可用结果，才叫真正装好。

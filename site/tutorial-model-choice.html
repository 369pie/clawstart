ClawStart 模型配置

模型选择与 API 配置

如果你卡在"到底先选哪个模型服务商"，这页就是给你的。先选最容易跑通的，跑通之后再考虑换更强的。
API Key 就是模型服务商给你的一串专属密码，用它来让 AI 助手连接模型。

推荐顺序 中国大陆普通用户第一次优先走：国内在线模型 → 已有海外 API 的模型 → 本地模型。先求稳定可用，再考虑成本和高级玩法。

第一步

先判断你属于哪一种情况

我只想尽快用起来

优先选国内在线模型，最省心，最适合第一次配置和验证连通性。

最推荐

我已经有海外 API Key

可以继续用你熟悉的服务商，但要先确认当前网络和账单都正常。

已有资源

我更想省钱或离线用

后续可以再切到本地模型，比如 Ollama，但不建议把它作为第一次必走路径。

后续再上

第二步

第一次优先选这 3 条路径之一

Path A
国内在线模型

适合想最快跑通的人

国内用户第一次主推这条，连通性和拿到 Key 的概率通常更高。

Path B
海外在线模型

适合已有账号和额度

如果你本来就有稳定的海外 API 资源，这条路径会更顺手。

Path C
本地模型

适合第二阶段再尝试

它适合后续控成本和离线使用，但不适合作为第一次主路径。

🇨🇳

国内在线模型

适合大多数中国大陆用户。拿到 API Key 相对容易，网络和连通性也更稳定。

  • 豆包 / 通义千问 / DeepSeek / 硅基流动
  • 适合第一次跑通
  • 优先用于首启验证
🌍

海外在线模型

适合已经有 OpenAI / Anthropic 等海外模型服务的用户，但要确认网络、余额和可访问性。

  • 适合已有账号和 Key 的用户
  • 需要网络可访问
  • 遇错先查 API 和代理
🏠

本地模型

适合后续想控成本、要本地化或愿意折腾环境的用户，不推荐作为第一次主路径。

  • 常见方案：Ollama
  • 需要本地硬件资源
  • 适合第二阶段再尝试

第三步

API Key 配置时，先检查这 4 件事

  • 复制时有没有带上前后空格
  • 这个 Key 是否还有效，是否有额度或余额
  • 当前网络是否能访问对应服务商
  • 你选择的模型服务商和填入的 Key 是否对应
Check 1
Key 输入框

先排除复制错误

最常见的问题不是服务商坏了，而是复制时多了空格、少了字符，或填错了对应平台的 Key。

Check 2
测试连接结果

优先看是不是网络问题

如果 Key 明明没错，但连接一直失败，很多时候是当前网络环境无法连接到模型服务器。

Check 3
第一次正常回复

跑通标准是得到稳定回复

别用“最强模型”当成功标准，第一次能稳定返回内容，才是现在最重要的指标。

第四步

第一次成功标准，不是“模型最强”，而是“连接稳定”

1

测试连接能稳定通过

2

能发出一个问题并正常得到回复

3

你知道当前这条模型路径以后还能继续用

什么时候该换模型

这些情况出现时，再考虑切到下一条路径

  • 当前模型连通性长期不稳定
  • 响应速度明显过慢，影响日常使用
  • 成本开始高于你的接受范围
  • 你已经能稳定使用，才考虑再切到本地模型
遇到 API Key 或网络报错怎么办 如果你已经选好了服务商，但测试连接还是失败，就直接去 排障页 看“模型连不上 / API Key 不通过”这一组，不要盲目反复换服务商。

ClawStart Skills 实操

安装第一个 Skill

第一次不要追求装很多。先选一个马上能用到的 Skill，完整走通搜索、安装、验证这条链路。

这页适合谁 适合已经能正常对话、想第一次安装 Skill，但不知道从哪开始的用户。

不想整页看完，就直接走这 4 个入口

如果你现在只想知道下一步该做什么，直接跳到对应步骤，不用从头慢慢滑。

先选 Skill → 直接看安装 → 直接看验证 → 装失败怎么办 →

先做一个选择

选一个你一周内会真正用到的 Skill

  • 想快速看到结果：优先选搜索 / 查询类
  • 想提高办公效率：优先选文件 / 内容整理类
  • 想辅助开发：优先选 Git / 代码相关类

步骤 1

先搜索，再判断

# 第一步不是先硬搜命令，而是先确认 Skill 的唯一名称（slug） # 打开 Skills Hub，看中文分类和真实 Skill 页面 # 选好以后直接执行安装命令 npx clawhub@latest install weather # 或者安装 Git / 仓库类 Skill npx clawhub@latest install github
判断原则 第一次优先选"名字直白、用途明确、你今天就能试一次"的 Skill，简单的用顺了再看组合型 Skill。
先去看 适合小白 + 无需 API Key → 打开完整 Skills Hub →
决策卡 先判断

先用这 4 个问题判断它值不值得装

  • 这个 Skill 的用途我能一句话说清吗
  • 我今天或这周内会至少用到它一次吗
  • 装上后，我能想到一个最短验证问题吗
  • 它能替我减少重复劳动，而不是只是看起来很酷吗

步骤 2

只装一个

Step 1
看到搜索结果

先挑名字直白的结果

第一次别选描述抽象的 Skill，先选你一眼就知道它是干什么的那一个。

Step 2
看到安装完成

安装成功只是中间状态

真正的目标不是“它出现在列表里了”，而是你接下来能立刻用一句话验证它。

Step 3
看到第一次调用结果

能调用，才说明值得留

只要能稳定完成一次调用，它才真正开始从“看起来有用”变成“我以后还会用”。

# 示例：安装 weather npx clawhub@latest install weather # 或安装百度搜索类 Skill npx clawhub@latest install baidu-search # 或安装 Git / 仓库类 Skill npx clawhub@latest install github
成功标准：命令执行完成后，你能立刻在对话里问一个最短问题来验证它。

步骤 3

什么时候不值得继续装

出现这些信号，就先换一个更直接的 Skill

  • 你看了名字和描述，还是不知道它到底帮你做什么
  • 你想不到任何一个今天就会问的问题
  • 它明显需要复杂前置条件，而你现在还在入门阶段
  • 你只是因为“别人说好像很强”才想装，而不是自己真会用

第一次装 Skill 的目标不是“装成专家”

你只需要完成一次搜索、一次安装、一次验证，这就已经比“知道有 Skill 但从没动手”更进一步了。

看新手推荐 Skill → 打开 Skills Hub → 再看工作流模板 → 回 Skills 指南 → 了解 LolaClaw →

新手入门

你现在最需要的不是学会所有功能，而是先完成第一次真正有结果的体验。

这页适合谁 适合已经完成下载和首次启动，但还不确定“接下来做什么”的用户。

第一次对话

确认它真的能理解你、回答你，而不是只停留在“看起来装好了”。

第一次任务

让它替你完成一个真实动作，最好是写作、整理、总结这类立刻能看出价值的事。

第一次成功标准

知道什么叫“已经跑通”，这样你后面扩展 Skill 或工作流时才不会发虚。

开始前准备

先确认这 4 件事

  • 应用能正常打开，不会闪退或被系统拦截
  • 模型连接已经测试成功，可以正常返回内容
  • 你知道当前工作区在哪里
  • 你准备好了一个很小、很具体的第一任务

步骤 1

先完成第一次对话

第一轮不要追求复杂，先问一个你能判断对错的问题，确认它已经进入“可用”状态。

# 示例 1：信息解释 解释一下 Docker 和虚拟机的区别 # 示例 2：短文本输出 帮我写一段 80 字以内的会议纪要开头 # 示例 3：整理类任务 把下面这段文字改写成更自然的口语风格
成功标准：你已经能稳定得到一条和你提问相关、可读、可判断的回复。

步骤 2

再做一个真实的小任务

第一次任务最好满足两个条件：很小、能马上看到结果。这样最容易建立“以后我还会继续用它”的感觉。

写作场景

比如写一段简介、润色一封邮件、整理一页内容提纲。

看任务模板 →

办公场景

比如总结一段会议记录、提炼一篇文章重点、输出待办清单。

看任务模板 →

开发场景

比如让它解释一段报错、写一个小脚本、检查 README 结构。

看任务模板 →

查看第一次任务模板 → 看资源补充 →

步骤 3

知道什么叫“已经跑通”

1

你能稳定打开 ClawStart

2

你能发出问题并收到正常回复

3

你已经完成了一个真实小任务

4

你知道接下来要么装 Skill，要么学工作流，而不是继续盲试

什么时候该去排障 如果你连第一次对话都无法完成，或者模型连接一会儿能用一会儿不能用，就先去 排障页，不要急着继续往下学。

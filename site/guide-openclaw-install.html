OpenClaw 中文安装指南

先选对路径，再去安装

如果你搜的是“OpenClaw 怎么装”，最重要的不是先看概念，而是先判断你是 Windows、macOS 还是 Linux 用户，再走最短路径。

给小白的结论 普通用户优先走 Windows 或 macOS 一键包。Linux 当前推荐通过安装脚本配置，更适合熟悉终端和愿意自己排错的用户。

第一步

先判断你属于哪条安装路径

🪟

Windows 用户

最适合普通用户和小白。直接下载一键包，按平台页和快速开始走就行。

看 Windows 路径 →
🍎

macOS 用户

支持 Apple Silicon 和 Intel。优先走一键包路径，最简单省心。

看 macOS 路径 →
🐧

Linux 用户

当前主推荐路径是 Linux 安装脚本。适合有终端经验的用户。

看 Linux 路径 →

第二步

最短安装链路，只记这 4 步

1

去下载页拿对的一键包

直接去 ClawStart 下载页，按平台拿到当前最新的安装包或脚本，这是最快的起点。

2

解压 / 打开安装包

按系统提示完成第一次打开。遇到 SmartScreen 或“无法验证开发者”时，不要慌，平台页和排障页都有说明。

3

完成第一次模型配置

第一次不要追求最强模型，先保证能稳定连接并拿到第一条正常回复。

4

回快速开始验证成功标准

能正常打开、能测试连接、能完成一次真实小任务，才算真正装好了。

第三步

这 4 个误区最容易让人白白浪费时间

  • 还没看下载页，就直接搜旧版命令行教程
  • 第一次就想同时折腾本地模型、API、Skills 和工作流
  • 装好以后不做验证，结果以为自己成功了，其实还没跑通
  • 遇到系统拦截就反复重下，不先看平台页和排障说明
什么时候该去排障 如果你已经拿到了正确平台的一键包，但应用打不开、模型连不上、Gateway 启动失败，就别继续乱试，直接去 排障页。

最推荐的下一步

先下载，再按快速开始完成第一次配置，不要在“到底怎么装”这一步停太久。

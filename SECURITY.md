# Security Policy

## Supported scope

Security-sensitive areas in this repository mainly include:

- installer scripts
- release workflows
- download links
- local execution paths

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem that could put users at risk.

Instead:

1. prepare a short summary of the issue
2. include affected files or flows
3. include reproduction steps if safe
4. share the report privately with the maintainers once a private contact address is published

Until a private contact is published, use your best judgment and avoid posting exploit details publicly.

## What to include

Helpful security reports usually include:

- affected file or workflow
- potential impact
- attack preconditions
- reproduction steps
- suggested mitigation if known

## Disclosure expectations

Please give maintainers reasonable time to assess and fix a vulnerability before public disclosure.
